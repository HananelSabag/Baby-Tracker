## Critical findings (security)

1. **`push_notification_log` is world-readable through PostgREST.** The
   `service_role_all_log` policy is bound to the `public` role with
   `USING (true)`. Anyone with the publishable anon key can read or mutate
   this table, leaking family/child relationships. Fix:
   `supabase/migrations/20260426_001_lock_down_push_notification_log.sql`.
2. **Avatars bucket allows anonymous listing**, and INSERT/UPDATE policies
   permit any authed user to write anywhere in the bucket. Fix:
   `supabase/migrations/20260426_002_avatars_bucket_no_listing.sql`.
3. **Personal Access Token in `.mcp.json`** ŌĆö gitignored, but live on disk.
   Rotate when not in active use.
4. **Hard-coded admin/wife/sister emails + `ADMIN_PASSWORD = '123456'`** in
   the client bundle. The actual admin route is protected by RLS server-side,
   but these literals are still privacy-leaky and confusing. Move out.
5. **`families_insert` is `WITH CHECK (true)`** ŌĆö any authed user can spam
   families. Add `created_by = auth.uid()` and rate-limit at the edge.
6. **Leaked-password protection disabled** in Supabase Auth settings.

## Real bugs in app code (fixed in this commit)

- **HomePage avatar upload wrote to a non-existent `members` table.** The
  silent catch hid the failure. Now writes to `family_members` and surfaces a
  toast.
- **`useState(() => {ŌĆ”})` used as `useEffect`** in `FamilyPage`. Fixed.
- **Avatars orphaned** because the storage path used the original file's
  extension. Now stable per-subject path with upsert.
- **Silent-fail on avatar upload.** Now surfaces toasts.
- **No image compression.** Now resizes to 1024px longest edge, JPEG q=0.85.
- **No camera capture.** New `PhotoSourceSheet` offers explicit
  "Take photo" and "Pick from gallery" actions.
- **Notifications dropdown anchored on the wrong side in RTL.** Now `right-0`.
- **Cluttered home header.** Bell + edit-view + avatar are now equal-size
  horizontal siblings; kid card avatar is 56px (was 32px).

## Open follow-ups (not fixed)

- Reports DOSE compliance "X / 14" doesn't account for partial weeks.
- Sleep-pair logic silently drops orphan starts/ends.
- HomePage `saveAndExit` `Promise.all` is partial-write risky on errors.
- `createFamily` console.log statements leak PII.
- Reports + History could batch-fetch related rows instead of using PostgREST
  embedded resources for every page render.

## Apply the migrations safely

Supabase branch first**, smoke-test the avatar upload + push pipeline, then
merge. Never run `db reset` against prod.
